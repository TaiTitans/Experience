## **Áp dụng quy trình giải bài toán**

Để tối ưu thời gian làm bài, hãy dùng **quy trình 4 bước**:

### 🔍 **Bước 1: Phân tích đề bài (Understand the problem)**

- Đọc kỹ đề, xác định **input**, **output**, **ràng buộc**.
- Xác định dạng bài toán: **Tìm kiếm? Tối ưu hóa? Đếm? Sắp xếp?**

### 🧠 **Bước 2: Tìm cách giải (Plan the approach)**

- Bắt đầu với cách **Brute Force (dễ hiểu nhưng chậm)**.
- Cải tiến từng bước để tìm thuật toán **tốt hơn**.
- Tự hỏi: **Có cách dùng HashMap / Two Pointers / Binary Search không?**
- Viết pseudo-code trước khi code.

### 💻 **Bước 3: Viết Code (Implement the solution)**

- Code sạch, dễ đọc.
- Kiểm tra với test case cơ bản trước.
- Nếu chạy sai, **debug bằng print hoặc debug tool**.

### ⚡ **Bước 4: Tối ưu hóa (Optimize the solution)**

- Phân tích **độ phức tạp thời gian O(n), O(log n), O(n²)**.
- Nếu bài toán quá chậm, thử **memoization, binary search, data structures khác**.
- Hỏi: **Có thể dùng Dynamic Programming không?**

## **Luyện tư duy thuật toán**

Để giỏi DSA, bạn cần **luyện tư duy giải bài toán**, không chỉ code.

📌 **Phương pháp tư duy:**

1. **Giải thích thuật toán bằng lời**:
    
    - Hãy tưởng tượng **bạn đang dạy người khác** (hoặc chính bạn trong tương lai).
    - Viết lại lời giải bằng tiếng Việt / tiếng Anh trước khi code.
2. **Tự hỏi & phản biện:**
    
    - Nếu bài toán quá chậm: **Có cách nào nhanh hơn?**
    - Nếu dùng quá nhiều bộ nhớ: **Có cách giảm O(n) về O(1) không?**
    - Nếu bài toán có vòng lặp lồng nhau: **Có thể dùng Binary Search hoặc DP không?**
3. **Code lại bài toán mà không nhìn code cũ**:
    
    - Sau khi giải, hãy thử code lại **từ đầu sau 2-3 ngày**.
    - Nếu vẫn nhớ cách làm, nghĩa là bạn đã thực sự hiểu bài toán.

## **Duy trì thói quen học tập**

🔥 **Lời khuyên quan trọng**:

- **Mỗi ngày làm ít nhất 1 bài** (hoặc 3 bài/ngày nếu rảnh).
- **Học theo cách chủ động**: Ghi chú công thức, thử code lại mà không nhìn đáp án.
- **Luyện tập đều đặn trong 3-6 tháng**, bạn sẽ thấy kỹ năng cải thiện rõ rệt.
